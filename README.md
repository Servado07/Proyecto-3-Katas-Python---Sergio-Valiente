Este repositorio contiene la resolución técnica del **Proyecto 3 de Python**, enfocado en el dominio de la lógica de programación, el paradigma funcional y la programación orientada a objetos (POO). A través de 40 desafíos (Katas), se exploran habilidades de algoritmia, manipulación de estructuras de datos y gestión de excepciones.

---

## 🎯 Objetivo del Proyecto

El propósito de este proyecto es consolidar conocimientos avanzados de Python mediante la resolución de ejercicios progresivos. Se abarca desde la manipulación básica de listas y cadenas hasta la creación de sistemas complejos de gestión bancaria y procesamiento de texto, garantizando un código limpio, eficiente y profesional.

---

## 🛠️ Instalación y Configuración

Para ejecutar este entorno de trabajo y validar las soluciones propuestas, se utilizó la siguiente configuración:

* **Lenguaje**: **Python 3.12.3**.
* **Entorno**: **Jupyter Notebook** para el desarrollo iterativo y pruebas.
* **Herramientas**: **VSCode** para la organización del código y **Git** para el control de versiones.
* **Librerías**: `math` para cálculos geométricos y `functools` para operaciones de reducción.

## 📊 Ejercicios y Retos Realizados

A continuación, se presenta un resumen de los hitos técnicos más relevantes resueltos en el archivo `proyecto3-katas-Python-SergioValiente.ipynb`:

| # | Desafío Técnico | Concepto Python Aplicado |
| :--- | :--- | :--- |
| **01** | Análisis de frecuencia de caracteres | Diccionarios y bucles `for` |
| **02** | Transformación masiva de datos | Función `map()` y `lambda` |
| **03** | Filtrado de elementos prohibidos | Función `filter()` |
| **04** | Reducción de listas a valores únicos | Función `reduce()` |
| **05** | Enmascaramiento de datos sensibles | Slicing y manipulación de Strings |
| **06** | Validación de anagramas complejos | Ordenamiento (`sorted`) y lógica booleana |
| **07** | Gestión de errores en búsquedas | Excepciones personalizadas (`raise`) |
| **08** | Sistema de crecimiento botánico (**Clase Arbol**) | Programación Orientada a Objetos (POO) |
| **09** | Simulador de banca y transferencias | Interacción entre objetos y `try/except` |
| **10** | Procesador de texto dinámico | Argumentos variables (`*args`) |
